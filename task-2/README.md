# Profile Card - Block vs Inline Elements

## 📌 Task Overview
This project demonstrates the difference between **block-level elements** and **inline elements** using a simple profile card layout in HTML and CSS.

## ✅ What I Implemented
- Basic HTML structure (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`)
- A profile card created using `<div>` (block-level element)
- Heading (`<h3>`) and paragraph (`<p>`) for content
- Inline elements inside the paragraph:
  - `<span>` for highlighting text
  - `<a>` for clickable links
- Styling using inline CSS:
  - Border, padding, and background color for the card
  - Center alignment using flexbox
  - Styled inline elements with different colors and font weight

## 🧩 Key Concepts Practiced

### 🔹 Block-Level Elements
- `<div>`, `<h3>`, `<p>`
- Start on a new line
- Take full width
- Stack vertically

### 🔹 Inline Elements
- `<span>`, `<a>`, `<b>`
- Do not start on a new line
- Only take as much width as needed
- Stay within the same line

## Experiment Performed
- Changed `<span>` (inline) to a block element like `<p>` / `<div>`
- Observed:
  - Text moved to a new line
  - Layout structure changed
- Restored `<span>` to understand when inline elements are preferred

## Learnings
- Block elements are useful for layout and structure
- Inline elements are used for styling or modifying parts of text
- Understanding both helps in designing better web layouts
