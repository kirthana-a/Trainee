# 📄 Semantic Blog Page – HTML Practice

## 📌 Overview
This project demonstrates how to convert a webpage built with generic HTML elements (`<div>` and `<span>`) into a properly structured page using **semantic HTML tags**.  

Semantic elements improve:
- Accessibility (screen readers understand structure)
- SEO (search engines better index content)
- Code readability and maintainability

---

## 🎯 Objective
The goal of this exercise is to:
- Replace non-semantic elements with meaningful tags
- Organize content logically using HTML5 semantic elements
- Understand the importance of document structure

---

## 🧱 Semantic Tags Used

| Tag | Purpose |
|-----|--------|
| `<header>` | Defines the top section of the page (branding/navigation) |
| `<nav>` | Represents navigation links |
| `<main>` | Wraps the primary content of the page |
| `<article>` | Represents individual blog posts |
| `<section>` | Groups related content within the page |
| `<aside>` | Contains related or sidebar content |
| `<footer>` | Defines the bottom section (copyright info) |


---

## 📝 Features Implemented
- ✅ Structured blog layout using semantic elements  
- ✅ Multiple articles inside the main content area  
- ✅ Sidebar for additional information  
- ✅ Navigation menu in header  
- ✅ Footer with copyright  
- ✅ Styled layout using CSS (Flexbox)
